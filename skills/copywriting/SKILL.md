# Structured Copywriting

## Core Principle

Every piece of copy answers one question: "Why should I care?" Answer it in the first sentence, then prove it.

## Structure: HPSP-C

### Hook
- First sentence grabs attention. State the benefit or the problem.
- BAD: "This library provides utilities for..."
- GOOD: "Stop writing the same validation logic in every project."

### Problem
- Why does the reader need this? What pain does it solve?
- Be specific about the cost of the status quo

### Solution
- What it does — concrete, not abstract
- Show, don't tell: code examples, before/after, screenshots

### Proof
- Evidence it works: benchmarks, adoption numbers, testimonials, comparison
- "Used by X teams" or "Reduces build time by 40%"

### Call-to-Action
- ONE clear next step. Not three options — one.
- "Run `npm install X` to get started" or "Read the quick start guide"

## Format-Specific Rules

| Format | Hook Style | Length | CTA |
|--------|-----------|--------|-----|
| **README** | Problem statement | 200-500 words | Install command |
| **Release notes** | Most impactful change | Bullet list | Upgrade instructions |
| **Blog post** | Surprising insight | 800-1500 words | Try it yourself |
| **Project description** | One-line value prop | 1-2 sentences | Link to repo |

## Anti-Patterns

- **Feature dump**: listing 20 features without explaining why any of them matter
- **Buried value prop**: 3 paragraphs before you learn what the tool does
- **No next step**: reader is convinced but doesn't know what to do next
- **Buzzword soup**: "leveraging AI-powered synergies for cloud-native paradigm shifts"
- **Passive voice**: "Files are processed by the system" → "The system processes files"
