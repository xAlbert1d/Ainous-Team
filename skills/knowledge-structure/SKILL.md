# Knowledge Structuring

## Core Principle

Structure serves retrieval. Organize so that someone with a question can find the answer in ≤3 navigation steps.

## Technique

### Step 1: Identify Categories
- Read ALL input before categorizing (don't categorize on first pass)
- Use MECE principle: Mutually Exclusive (no overlap), Collectively Exhaustive (nothing missing)
- Test: does every item fit in exactly ONE category? If not, categories overlap.
- Max 7 top-level categories (cognitive limit for navigation)

### Step 2: Build Hierarchy
- Max 3 levels deep. Deeper hierarchies become navigation mazes.
- Level 1: topic areas (≤7)
- Level 2: subtopics (≤5 per topic)
- Level 3: specific facts/entries (unlimited but indexed)
- If a category has only 1 item, merge it up

### Step 3: Add Cross-References
- Identify items that relate to multiple categories
- Don't duplicate — reference: "See also: [other section]"
- Cross-references are evidence of a well-structured knowledge base

### Step 4: Validate Completeness
- For each category: is anything missing that you'd expect to find here?
- For each item: is it in the right category?
- The "new person" test: could someone unfamiliar navigate this to answer a real question?

## Frameworks

| Framework | Best For | Structure |
|-----------|----------|-----------|
| **MECE** | Business analysis, exhaustive categorization | Non-overlapping, complete |
| **Concept map** | Learning, exploration | Nodes + labeled relationships |
| **Decision matrix** | Choosing between options | Options × criteria grid |
| **Ontology** | Domain modeling | Is-a / has-a / part-of relationships |
| **Timeline** | Historical context, project evolution | Chronological with milestones |

## Anti-Patterns

- **Premature categorization**: creating the taxonomy before seeing all the data. Read first, structure second.
- **Overlapping categories**: "Tools" and "Technologies" will confuse everyone. Pick one.
- **Too-deep hierarchies**: 5 levels deep means 5 clicks to find anything. Flatten.
- **Orphan items**: items that don't belong to any category — either create a category or question whether the item belongs
- **Category bloat**: 20 top-level categories is a flat list pretending to be structured
